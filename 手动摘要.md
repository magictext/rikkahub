 ---
  功能概述

  为聊天中的每条 LLM
  响应增加手动生成摘要的功能，同时在压缩历史区域增加批量摘要功能。摘要使用的模型可以在模型设置中独立配置。

  具体改动

  1. 模型设置层

  - 在 Settings 数据类中新增 summaryModelId 和 summaryPrompt 字段
  - 在模型设置页面新增一个"摘要模型"配置卡片，与现有的"压缩模型"卡片样式一致，支持选择模型和自定义 prompt

  2. 后端逻辑

  - ChatService 中新增 generateMessageSummary() 方法：对单条消息调用 LLM 生成摘要，将摘要作为 UIMessagePart.Summary
  嵌入到消息的 parts 中
  - ChatService 中新增 batchGenerateSummaries() 方法：遍历所有没有摘要的助手消息，批量调用 LLM 生成摘要并嵌入
  - ChatVM 中新增对应的两个 handler 方法

  3. 单条消息摘要按钮

  - 在每条助手消息的操作按钮区域（FlowRow）中，增加一个"生成摘要"图标按钮
  - 如果消息已有摘要则隐藏该按钮
  - 按钮点击后调用 ChatVM.handleGenerateMessageSummary(messageId)

  4. 批量摘要按钮

  - 在 FilesPicker（附件选择器）中，"压缩历史"按钮下方增加一个"批量生成摘要"列表项
  - 显示没有摘要的消息数量作为提示
  - 点击后调用 ChatVM.handleBatchGenerateSummaries()

  5. 回调链路

  - onGenerateSummary 回调从 ChatMessage -> ChatList -> ChatPageContent -> ChatVM 逐层传递
  - onBatchSummary 回调从 FilesPicker -> ChatInput -> ChatPageContent -> ChatVM 逐层传递

  6. 字符串资源

  - 新增中英文等多语言字符串资源，包括设置页面标题、按钮文案、错误提示等

  ---
  核心思路：摘要生成复用现有 generateTitle() 的模式——通过 Provider.generateText() 进行简单的非流式 LLM
  调用，将返回的文本封装为 UIMessagePart.Summary 并追加到消息的 parts 列表中。
